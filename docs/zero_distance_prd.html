产品定位

妍叶之庭 YanYe Home 是一个“情侣生活操作系统”，核心不是记录爱，而是让两个人更容易把爱落实到行动里。

目标平台：Android 优先。后续可扩展为 PWA 或微信小程序。视觉方向为日系极简、干净、无广告、轻动效、重隐私。

核心人群

长期稳定、互动密度高的情侣

两人关系基础深，已经有大量共同记忆、内部笑点和未来规划。App 应该承接真实生活，而不是制造表演型打卡。

核心痛点

不是没爱，是生活太碎

吃什么、去哪玩、纪念日、旅行回忆、承诺兑现、冷战破冰都分散在聊天记录里，时间久了很难检索，也难以沉淀。

核心价值

减少消耗，增加留下来的东西

把容易反复争执的事项产品化，把值得珍惜的瞬间结构化，让 App 每次打开都能帮忙做一件具体有用的事。

产品原则

本产品必须克制。每个功能都要回答一个问题：它是否能被反复使用，是否能和已有数据产生联动，是否能让关系变轻一点。

1. 少模块，多闭环 不做孤立功能。愿望清单、日程、地图、挑战、纪念日必须互相生成内容。
2. 轻输入，重回看 日常记录不能太麻烦。优先使用选择、模板、照片、标签，而不是长篇填写。
3. 隐私默认安全 吵架记录、私密愿望、信箱、健康记录、边界相关内容默认只有创建者可见。
4. 关系修复优先 冲突模块不用于审判谁错，而是帮助降温、复盘、兑现承诺。

核心使用闭环

App 的主线是“计划一次共同体验，完成后沉淀为回忆，回忆再反哺下一次计划”。

这条闭环保证功能不是堆砌：每个模块都能把数据传给下一个模块。

灵感进入

愿望清单、随机挑战、今天吃什么、纪念日提醒产生一个可执行想法。

变成日程

双方确认时间后，自动进入情侣日历，并附带预算、地点、准备事项。

一起完成

当天 App 只展示和这件事有关的信息：路线、餐厅、任务、拍照入口。

沉淀回忆

照片、花费、城市、省份、美食、心情、梗自动归档到时间线和地图。

再次唤起

纪念日前、吵架后、周末前，App 从旧回忆里推荐可复刻的小计划。

核心功能需求

以下保留了最初提出的必需功能，并从问卷中筛选出能长期产生价值的增强能力。

优先级：P1 为首版必须实现，P2 为首版可简化实现，P3 为后续增强。

P1 · 首页

今日面板

所有功能的入口不直接平铺，而是通过今日状态聚合。

需求描述

首页展示今天最该被看见的内容：最近纪念日、当天日程、待确认愿望、吃什么入口、双方心情状态、下班提醒、未完成承诺。避免用户每次打开都面对一堆模块图标。

能力 说明 优先级
今日卡片 自动聚合纪念日、日程、愿望、挑战、冲突冷却状态。 P1
状态灯 同步“开心、累、想贴贴、需要独处、在忙”等轻量状态。 P1
下班打卡提醒 到预设下班时间弹窗提醒，一键发送“下班啦”“在路上”“今天要加班”。 P1
快捷行动 一键进入“吃什么”“新增日程”“记录回忆”“冷静模式”“新备忘”。 P1
P1 · 纪念日

纪念日与倒数日

必须功能。承担提醒、回忆和仪式感。

需求描述

支持添加恋爱纪念日、生日、节日、第一次旅行、定情信物日期等。每个纪念日不仅是倒数，还能绑定礼物愿望、餐厅候选、历史照片和去年如何庆祝。

能力 说明 优先级
倒数与周年 显示距离某日还有几天、已经在一起多久、第几个周年。 P1
礼物联动 纪念日前自动推荐双方公开或私密愿望清单中的礼物灵感。 P1
庆祝档案 记录每年吃了什么、送了什么、去了哪里、拍了哪些照片。 P2
P1 · 愿望

愿望清单

必须功能。支持共享与仅自己可见。

需求描述

愿望不是普通备忘录，而是未来日程、礼物和挑战的灵感池。分类包含购物、游玩、餐厅、礼物、家具、养狗准备、共同学习、必看必玩等。

能力 说明 优先级
可见性 每条愿望可设为双方可见、仅自己可见、指定日期后可见。 P1
分类与标签 支持购物、游玩、餐厅、礼物、家具、养狗、电影、游戏等分类。 P1
转日程 愿望确认后可一键生成日程，自动带入地点、预算、准备事项。 P1
快乐预算提示 针对游戏充值等即时消费设置个人快乐预算，只做温和提醒。 P3
P1 · 日程

情侣日历

必须功能。承接所有“要一起做的事”。

需求描述

共同日历用于记录约会、旅行、纪念日、演出活动、养狗准备、家务维修和共同任务。它不是商务日历，而是两个人的生活安排中心。

能力 说明 优先级
共同事件 标题、时间、地点、提醒、预算、参与人、关联愿望。 P1
攻略模式 适合你们的攻略型出游，事件下可添加餐厅、活动、路线、备选方案。 P2
完成归档 日程结束后提示补充照片、美食、花费、心情，并生成回忆卡。 P1
P1 · 地图

点亮地图

必须功能。省份级到城市级的足迹回忆。

地图不是打卡表，是回忆索引

全国省份地图展示已点亮省份，点击省份进入市级地图。每个城市可以绑定“吃了什么、玩了什么、照片、日程、梗、花费、避坑提示”。

北京麦当劳、演出、纪念日大餐
上海日料、展览、随机挑战
成都烧烤、酒店躺平、旅行复盘
能力 说明 优先级
省市二级点亮 中国省份级地图，点击进入城市级地图；支持手动点亮。 P1
旅行记忆 记录当地吃过的店、玩过的地方、避坑、照片、评分和日期。 P1
日程联动 旅行日程完成后自动询问是否点亮城市并生成城市回忆。 P1
P1 · 冲突

吵架记录与修复

必须功能。重点是降温和复盘，不是审判。

需求描述

你们的冲突特点是当面爆发、短暂冷战、通常男方先破冰，核心需求是“冷战期间不要断联”和“承诺要兑现”。因此冲突模块分为冷静模式、道歉小气泡、复盘记录、承诺墙四层。

能力 说明 优先级
冷静模式 一键进入低刺激界面，只保留状态投递：需要独处、可以抱抱、稍后聊。 P1
道歉小气泡 提供低压力破冰入口，可发送“我错了”“想抱一下”“我会改”“先不吵了”等可爱气泡。 P1
吵架复盘 模板化记录触发原因、真实情绪、希望对方下次怎么做、是否已经和好。 P1
承诺兑现墙 道歉后的承诺进入待兑现列表，完成后由对方确认，形成信任记录。 P2
敏感话题提醒 记录前任、特定虚拟主播、异性边界等高敏话题，只在复盘时可见。 P3
P1 · 决策

今天吃什么

必须功能。解决每天都会出现的真实问题。

需求描述

“今天吃什么”采用转盘与筛选结合。先根据忌口、距离、预算、营业时间、最近吃过次数过滤，再用转盘提供仪式感。核心规则必须记住：不推荐蘑菇；烧烤、日料、麦当劳、海鲜、猪肉类偏好可作为权重。

能力 说明 优先级
餐厅池 支持自建餐厅、菜系、忌口、价格、地址、是否踩雷。 P1
偏好过滤 按“不吃蘑菇”、想吃肉、想吃海鲜、是否外卖、是否出门过滤。 P1
转盘决策 筛选后再转盘，结果可一键变成日程或记录到城市美食档案。 P1
公平权重 统计近期谁妥协更多，下一次决策适度倾斜给对方偏好。 P3
P1 · 照顾

关怀备忘中心

承接备忘录、下班提醒、姨妈记录这些高频但不宜做重的需求。

需求描述

关怀备忘中心用于处理两个人每天会忘、但忘了会影响体验的小事：下班有没有回家、明天要带什么、姨妈期是否需要照顾、最近是不是怕冷或状态差。它不做复杂健康管理，而是做“亲密关系里的提醒与照顾”。

能力 说明 优先级
共享备忘录 支持共享或私密备忘，可设置提醒、清单、图片、关联日程，例如带伞、买药、取快递。 P1
下班弹窗提醒 按工作日和自定义时间弹窗，一键同步“下班啦”“晚点走”“到家了”，减少对方等待和担心。 P1
姨妈记录 记录周期、开始/结束日期、疼痛程度、情绪、忌口和照顾偏好；默认私密，可选择共享提醒。 P1
照顾建议 姨妈期或状态差时，首页给对方显示轻量提示，例如准备热水、别安排太累的行程。 P2
P2 · 回忆

时光信箱与梗百科

保留“时光信箱”，新增长期价值最高的梗库。

需求描述

时光信箱用于延时开启的信、照片、承诺和未来祝福。梗百科用于记录你们独有的黑话、冷笑话、儿童派瞬间、游戏和二次元记忆。它们共同承担“多年后再看仍然珍贵”的价值。

能力 说明 优先级
延时信件 写给未来某天的对方或双方，未到时间不可查看。 P2
梗百科 词条、来源、相关照片、关联人物、第一次出现时间、搜索。 P2
导出纪念册 后续支持按年份导出 PDF 或网页纪念册。 P3
P2 · 挑战

Adventure Scratch Off

必须功能。以刮刮乐呈现 50 个情侣挑战。

需求描述

预设 50 个小挑战，例如一起看日出、去对方小学门口吃零食、互相给对方搭一套穿搭、一起完成一次 AI 视频、寻找北京一家宠物友好咖啡店。刮开前只显示难度、预算、时长和天气需求，刮开后可接受、改期或放弃。

能力 说明 优先级
刮刮乐交互 刮开后显示挑战内容，支持接受并生成日程。 P2
完成认证 完成后上传照片、文字、地点，可点亮挑战徽章。 P2
个性化挑战池 结合你们的偏好：动物、游戏、二次元、烧烤日料、北京生活、养狗准备。 P3

关键联动设计

为了避免“没用模块集合”，首版至少要实现这些跨模块联动。

联动越自然，用户越不需要记住有哪些功能。

愿望 → 日程 → 回忆

愿望不是收藏夹

愿望被双方确认后生成日程；日程完成后自动归档为回忆；如果包含地点，则同步到点亮地图。

纪念日 → 礼物 → 私密愿望

礼物不靠猜命

纪念日前自动提示“可公开愿望”和“仅自己可见的礼物计划”，既有惊喜感，也不会完全失手。

吃什么 → 餐厅档案 → 城市地图

每顿饭都能沉淀

转盘选中的餐厅吃完后可评分、记录菜品与避坑；如果在旅行城市，则同步到城市回忆。

吵架 → 承诺 → 今日面板

复盘要有后续

复盘里产生的承诺进入今日面板待办。完成后由对方确认，减少“说了但没做到”的二次失望。

挑战 → 日程 → 徽章

随机性必须可执行

刮开挑战后必须能直接安排时间、添加准备事项，完成后点亮徽章并写入年度回忆。

状态灯 → 冷静模式

低成本表达

当一方设置“需要独处”或“想被抱抱”，另一方首页同步展示，避免因为不说话产生额外误会。

姨妈记录 → 今日面板 → 日程建议

照顾要能落地

姨妈期可在首页给对方轻提示，并在安排日程或随机挑战时降低强度，优先推荐室内、近距离、少走路方案。

下班打卡 → 状态灯 → 吃什么

从“下班了吗”到“吃什么”

下班弹窗同步状态后，可顺手进入吃什么或晚间日程，减少反复问询，也让见面安排更自然。

数据结构要围绕“事件”和“回忆”设计，而不是围绕页面设计。

首版可使用本地数据库，后续增加双人同步与端到端加密。

MVP 版本规划

首版目标不是“一次做完所有浪漫想象”，而是做出一个每天或每周真的会打开的版本。

建议 6-8 周完成可用首版，后续按使用频率迭代。

V0.1 · 私密单机原型

实现纪念日、愿望清单、情侣日历、今天吃什么、基础备忘录、姨妈记录、基础回忆归档。数据本地存储，先保证你自己能完整跑通核心闭环。

V0.2 · 双人可用版本

加入双人账号、共享与私密可见性、状态灯、下班打卡提醒、冷静模式、道歉小气泡、愿望转日程、日程转回忆。

V0.3 · 地图与挑战版本

实现省市二级点亮地图、城市回忆、Adventure Scratch Off、完成后照片点亮。

V1.0 · 情人节礼物版本

完善视觉、动效、隐私加密、数据备份、纪念册视图，并预置你们专属的餐厅、梗、挑战和纪念日。

隐私与边界

这是情侣 App 最容易做错的部分。产品要增强安全感，而不是制造监控感。

默认尊重私人空间：可以分享，但不能强迫透明。

默认私密

敏感内容先保护

吵架复盘、私密愿望、时光信箱、礼物计划、姨妈记录、边界话题默认仅创建者可见，分享需要主动确认。

健康边界

姨妈记录由本人控制

周期、症状和情绪属于敏感健康数据。对方只能看到本人选择共享的提醒级信息，例如“今天不舒服”“别安排太累”。

不做监控

拒绝位置追踪与查岗依赖

不做常驻定位、不做强制拍照查岗。趣味互动可以存在，但必须可拒绝、可关闭、可追溯。

端到端加密

双人数据只属于两个人

后续云同步版本应使用端到端加密。服务端只保存密文，无法读取具体内容。

关系友好

冲突记录不做排行榜

不统计谁错得更多，不展示吵架次数排行榜。复盘只服务于理解、和好和承诺兑现。

不建议首版实现的功能

以下想法有趣，但首版容易增加复杂度或造成压力，建议暂缓。

心跳或呼吸震动同步

实现成本高，用户本人也认为必要性不足。可以用“想贴贴”状态灯替代。

复杂共同财务系统

你们目前财务只是部分重合，首版不需要做完整账本。保留大额预算、旅行花费记录即可。

社交边界报备系统

容易从安全感工具变成压力工具。可先在冲突复盘里记录边界共识，不做日常报备。

妍叶之庭 YanYe Home 的首要目标：让你们在忙、累、纠结、吵完架的时候，仍然更容易回到彼此身边。
